# LibreChat
LibreChat is a chatbot interface for the OpenAI API, the Anthropic API and others.
- https://www.librechat.ai/
- https://github.com/danny-avila/LibreChat


## API Pricing
- https://openai.com/api/pricing/
- https://www.anthropic.com/pricing#anthropic-api


## Setup

1. Use the [Credentials Generator](https://www.librechat.ai/toolkit/creds_generator) to generate the random keys and and enter them in the individual enviroment files:
   - `CREDS_KEY` -> `./config/env.librechat.ovr`
   - `CREDS_IV` -> `./config/env.librechat.ovr`
   - `JWT_SECRET` -> `./config/env.librechat.ovr` and ``./config/env.rag_api``
   - `JWT_REFRESH_SECRET` -> `./config/env.librechat.ovr`
   - `MEILI_MASTER_KEY` -> `./config/env.meilisearch`

Note: `./config/.env.librechat` is the original [template](https://github.com/danny-avila/LibreChat/blob/main/.env.example).
The file `./config/.env.librechat.ovr` is used to override individual values of the original template.


2. Customize the file `./config/librechat.yml` according to your needs.
    - For more information, see the [Configuration Guide](https://www.librechat.ai/docs/configuration/librechat_yaml)
    - [Template](https://github.com/danny-avila/LibreChat/blob/main/librechat.example.yaml)


3. Start the system with docker compose
```
docker compose up -d
```


## Create User
https://www.librechat.ai/docs/configuration/authentication#create-user-script

1. Execute container shell 
```
docker exec -it LibreChat sh
```
2. Run `create-user` script
```
npm run create-user
```


## Services
The overall system is made up of 5 services:
- [api](https://github.com/danny-avila/LibreChat): LibreChat frontend and backend
- [mongodb](https://hub.docker.com/_/mongo): NoSQL database to store user accounts and chat messages
  - Also see [MongoDB Authentication (Docker)](https://www.librechat.ai/docs/configuration/mongodb/mongodb_auth)
- [meilisearch](https://github.com/meilisearch/meilisearch): Local search engine to search through chat messages
- [vectordb](https://hub.docker.com/r/ankane/pgvector/tags): Vecotr DB for storing text embeddings generated by the `rag_api`
- [rag_api](https://github.com/danny-avila/rag_api): Framework for document indexing and retrieval
